**Resumen del Proyecto:**  
El proyecto **Agenda UC.CL** tiene como objetivo desarrollar una versión mejorada y actualizada de la anterior API de la Agenda UC. Consiste en una API diseñada para servir a toda la universidad y un sitio central que funciona como el núcleo del sistema de agendas.

## Objetivos

1. Desarrollar una API centralizada que sirva como sistema de agendas para toda la comunidad universitaria.
2. Proporcionar una experiencia de usuario mejorada y más eficiente en comparación con la versión anterior.
3. Asegurar la integración fluida entre el frontend y el backend para mejorar la usabilidad del sistema.

## Funcionalidades

1. **API Centralizada:**

- La API sirve como el punto central para gestionar todos los eventos y actividades relacionadas con la universidad, proporcionando datos actualizados para los usuarios.

2. **Desarrollo Frontend Personalizado:**

- El frontend del sitio se ha desarrollado utilizando **React** y **Next.js**, en combinación con el **Kit Digital UC**, para mantener la consistencia visual e institucional.

3. **Soporte y Mantenimiento Continuo:**

- El sistema se despliega en **Vercel** y se mantiene con actualizaciones regulares para garantizar un funcionamiento continuo y eficiente.

## Stack Tecnológico

- Backend (API):
  - **Node.js** (Strapi Headless CMS) para la gestión de contenidos.
  - **MySQL** como base de datos.
- Frontend:
  - **React** (Next.js) para el desarrollo del sitio.
  - **Kit Digital UC** (integrado vía NPM) para mantener la consistencia visual.
- Despliegue del Frontend:
  - **Vercel** para la infraestructura de despliegue y hosting.

## Resultados

El proyecto **Agenda UC.CL** se ha consolidado como un sistema eficiente y robusto para gestionar las actividades y eventos de la universidad. Mi participación fue clave en el desarrollo del frontend, colaborando con el equipo para asegurar una interacción fluida con el backend y contribuyendo al desarrollo del modelo de datos que mejora la usabilidad del sistema.
